# LLM 能做什么与不能做什么

> 吴恩达《Generative AI for Everyone》第四章学习笔记

## 能力边界：知之为知之

这一章的主题很实在——**LLM 擅长什么、不擅长什么、以及我们应该如何与之协作来弥补它的短板。** 吴恩达没有回避 LLM 的局限性，而是坦诚地画出了能力边界。这一点很重要，因为高估 AI 的能力往往会导致项目失败，而低估它又会错失机会。

## 擅长的事：LLM 的"舒适区"

LLM 在以下领域表现非常出色：

**文本生成与翻译**：这是 LLM 最基础也是最擅长的能力。从创意写作到多语言翻译，质量通常令人满意。

**Summarization（摘要）**：将长文本压缩为关键要点，LLM 的表现已经可以媲美人类。信息密度越高的文本（技术文档、研究论文），效果越好。

**Code Generation（代码编写）**：LLM 能生成符合语法的代码、解释代码逻辑、甚至进行简单的代码重构。这是目前最实用的开发者工具之一。

**Brainstorming & Ideation**：前面章节提到的 Thought Partner 能力——出点子、扩展思路、从不同角度思考问题。

**Stylistic Adaptation（风格转换）**：把同一内容写成学术风格、营销风格、口语风格——LLM 可以像变色龙一样切换风格。

## 不擅长的事：LLM 的"雷区"

**精确计算（Arithmetic）**：LLM 不是计算器。你问它 1234 × 5678 等于多少，它可能会给你一个"看起来合理但实际上是错的"答案。数学计算需要符号推理，而 LLM 做的是概率预测——它学的是"类似问题的人通常怎么回答"，而不是真的在"算"。

**实时信息（Real-time Knowledge）**：LLM 的知识截止于训练数据的时间点。它不知道今天的天气、最新的新闻、或者刚发布的某款产品的价格。要获取实时信息，必须通过外部工具（如联网搜索、API 调用）来补充。

**Hallucination（幻觉）**：这是 LLM 最棘手的问题之一。**模型会"编造"听起来合理但完全不正确的信息。** 它可能自信满满地告诉你"某某理论是谁在什么时候提出的"，而实际上那个人根本没做过那个研究。问题在于——它编造的文本和它给出的正确信息，在"表达方式"上几乎无法区分。这种"看起来极真"的幻觉，比一眼就能看出的错误更具迷惑性。

**复杂逻辑推理（Complex Reasoning）**：虽然 Chain-of-Thought（思维链）可以显著改善 LLM 的推理能力（通过让模型"一步一步思考"而不是直接给出答案），但它仍然不能保证 100% 正确。多步逻辑推理中的某一步出错，就会导致最终答案完全跑偏。

## 解决之道：外部工具 + 人工验证

吴恩达提出了一个非常务实的解决方案：**不要在 LLM 的短板上死磕，而是给它配上工具。**

- **计算器（Calculator）**：需要精确计算时，让 LLM 生成代码，用计算器执行，而不是让它直接"心算"。
- **搜索引擎（Search Engine）**：需要实时或事实性信息时，用 Retrieval-Augmented Generation（RAG）模式——先搜索，再让 LLM 基于搜索结果作答。
- **人工验证（Human Review）**：对于高风险场景（法律、医疗、财务），AI 的输出必须经过人工审核。

这里的核心原则是：**把 LLM 当作一个"产生建议的系统"，而不是"产生答案的系统"。** 建议可以很出色、很有创造性，但最终的决定和验证权仍在人类手中。

## Chain-of-Thought（思维链）技巧

CoT 是目前提升 LLM 推理能力最有效的方法之一。吴恩达举例说明：

❌ 直接提问："小明有 5 个苹果，小红有 3 个，小红比小明少几个？" → LLM 有可能答错。

✅ 引导思维链："小明有 5 个苹果，小红有 3 个。第一步：小明比小红多的数量是 5 - 3 = 2。所以小红比小明少 2 个。" → 正确率大幅提升。

CoT 的原理是让 LLM 的"思考过程"可见且分解，减少了模型跳过关键步骤而犯错的风险。虽然不能保证 100% 正确，但在推理类任务上改善非常明显。

## 个人理解

这一章让我印象最深的是对 Hallucination 的精确定义。**以前我觉得 LLM 犯错是因为它"不懂"，但现在我理解到——它其实是在"编小说"。** 模型没有"记忆"和"知识"的概念，它只有"什么词最可能接在后面"的概率分布。从这个角度看，幻觉不是 bug，而是 LLM 工作方式的必然产物。

另一个重要收获是：**不要试图让 LLM 做它不擅长的事。** 擅长生成和归纳 → 让它做这些；不擅长计算和实时知识 → 给它配工具。理解这个分工，远比幻想 LLM 是万能的更实际。

最后，"Human-in-the-loop"绝不是权宜之计——**在可预见的未来，最好的 AI 系统都是人机协作的系统，而不是完全自动化的系统。** 承认这个事实，才能设计出真正可靠的 AI 应用。
