# 写作、阅读、聊天 — 三大核心用例

> 吴恩达《Generative AI for Everyone》第三章学习笔记

## LLM 的"三头六臂"

吴恩达将 LLM 的核心能力精炼地归纳为三类：**Writing（写作）、Reading（阅读）、Chatting（聊天）**。虽然听起来简单，但这三个方向几乎覆盖了知识工作者日常工作中的绝大部分场景。

理解这三种模式，比记住一堆 API 参数更重要——它决定了你什么时候该用 LLM，以及怎么用才能发挥最大价值。

## Writing：不仅仅是从零生成

Writing 模式最容易理解——让 LLM 帮你写东西。但它远不止"帮我写一封邮件"这么简单。

实际应用场景非常丰富：

- **营销文案**：为产品写宣传语、公众号推文、广告文案
- **邮件撰写**：根据要点自动生成正式或友好的邮件草稿
- **报告起草**：给一个大纲，让 LLM 填充具体内容
- **创意写作**：故事梗概、角色设定、对话剧本

值得注意的是，吴恩达强调：**LLM 最适合的是"初稿"，而不是"终稿"。** 让 AI 先生成一个框架和主要内容，然后由人来修改润色——这比从零开始写快得多，也比让 AI 直接输出最终版本可靠得多。人机协作（Human-in-the-loop）是这个模式的最佳实践。

## Reading：信息过载时代的解药

Reading 模式可能比 Writing 更有用，因为我们每天接触的信息量远远超过我们能产出的量。LLM 在信息处理上的能力包括：

- **Summarization（摘要）**：把一篇长文压缩成几个关键要点
- **Classification（分类）**：把客服工单自动分到不同部门
- **Extraction（信息提取）**：从一篇客户评论中提取情感倾向、提及的具体问题
- **Q&A on Documents（文档问答）**：基于企业知识库回答问题

吴恩达举了一个很接地气的例子：假设你有一千条客户评论，想知道大家对某个新功能的态度。传统做法是让人工一个个看，费时费力。用 LLM，一条 prompt 就能在一分钟内完成情感分析和主题归类，还能生成一份结构化的报告。

这就是 **Prompt-based Development** 的核心魅力：**你不需要训练模型，你只需要写提示词。**

## Chatting：不只是聊天机器人

Chatting 模式覆盖了从简单的客服机器人到复杂的角色扮演场景。但吴恩达指出，现代 LLM 的 chatting 能力已经远超传统的"FAQ 问答"：

- **Customer Support（客服）**：结合企业知识库，让 AI 处理常见问题
- **Role-playing（角色扮演）**：让 LLM 扮演面试官来帮你练习求职面试
- **Tutorial Chat（教学对话）**：AI 作为 tutor，以对话形式解答学习中的疑问
- **Coaching（教练）**：让 AI 扮演一个"提问者"而不是"回答者"，帮你理清思路

一个有趣的洞察是：**Chatting 和 Writing 的核心区别在于上下文管理。** Chatting 需要记住前面说了什么、需要保持对话的一致性，这对 context window 和对话管理提出了更高要求。

## Prompt-based Development vs Supervised Learning

这是理解 GenAI 应用开发范式的关键区别：

**传统 Supervised Learning（监督学习）**：
- 需要数千到数百万条标注数据
- 需要数周或数月的模型训练
- 每次一个新任务就要重新训练
- 总成本：十万美元到数百万美元

**Prompt-based Development（基于提示词的开发）**：
- 只需几条精心设计的示例
- 不需要训练模型，直接使用现有的 LLM
- 同一个模型可以做无数不同的任务
- 总成本：几美分的 API 调用费

这个对比异常鲜明。**GenAI 真正革命性的地方，不是它"更聪明"，而是它让 AI 应用开发的门槛降低了几个数量级。** 以前只有大公司玩得起的 AI，现在一个独立开发者花一个下午就能集成。

## 个人理解

这一章给了我一个很实用的框架：**面对一个任务时，先问自己——这属于 Writing、Reading、Chatting 的哪一种？** 分类清楚了，prompt 的方向就清晰了。

另外，Prompt-based Development vs Supervised Learning 的对比，让我意识到一个更深层的趋势：**AI 的"民主化"正在真实地发生。** 技术门槛的降低意味着竞争的焦点将从"能不能做 AI"转向"用 AI 做出来的东西好不好"。创意、产品思维和领域知识，反而会成为更稀缺的能力。
