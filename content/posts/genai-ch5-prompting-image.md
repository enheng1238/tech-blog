# 第5章：提示工程技巧与图像生成

## 一、提示工程的核心原则

### 1. 清晰具体（Be Specific）

Prompt（提示词）的质量直接决定 LLM（大语言模型）的输出质量。Andrew Ng 给出的第一条原则是：**清晰且具体**。模糊的 prompt 得到模糊的回答。例如，与其说"写一封邮件"，不如说"写一封给客户的邮件，告知项目延期两周，语气专业且诚恳，附带补救方案"。

为什么？因为 LLM 本质上是一个概率模型，它根据你的输入来预测最可能的下一个 token。输入越模糊，模型的可选空间越大，结果越不可控。

### 2. 给模型思考时间（Give the Model Time to Think）

这是很多人容易忽略的一点。如果你让模型做复杂推理（如数学题、多步分析），直接问"答案是什么"往往得到错误结果。更好的做法是引导模型**逐步推理**——类似于思维链（Chain-of-Thought, CoT）。

例如，不要问："这栋楼需要多少块砖？"而是问："先计算建筑面积，再计算每平米所需砖数，最后考虑损耗率，请分步计算。"这种方式大幅提升了复杂任务的准确率。

### 3. 使用分隔符（Use Delimiters）

当 prompt 中包含多段内容（比如用户评论、长文本），使用分隔符可以帮助模型明确哪部分是"指令"，哪部分是"输入数据"。常见分隔符包括：

- 三重引号 `"""`
- XML 标签 `<tag></tag>`
- 三连反引号 `````

这不仅能提高准确性，还能有效防止 prompt injection（提示注入攻击）——当用户输入中包含恶意指令时，分隔符可以将其与系统指令隔离。

### 4. Few-Shot Prompting（少样本提示）

在 prompt 中给出几个示例，告诉模型"我想要的输出长什么样"。比如做一个情感分析，你可以给两三个"输入 → 情感标签"的例子。LLM 会从这些示例中推断出你想要的格式和判断标准。

- **Zero-shot**：不给示例，直接问
- **One-shot**：给一个示例
- **Few-shot**：给 2-5 个示例

Few-shot 在格式控制方面非常有效，尤其是输出格式需要严格遵循某种模板时。

## 二、图像生成原理

### Diffusion 模型——从噪声到图像

第5章深入介绍了当今图像生成的核心技术：**Diffusion Model（扩散模型）**。它的基本思想令人惊叹：先学习如何把一张图片逐步加噪变成纯噪声，然后反向——学会从纯噪声中逐步去噪，恢复出有意义的图像。

整个过程分为两个阶段：

1. **Forward（前向扩散过程）**：对训练图像逐步添加高斯噪声，直到完全变成随机噪声。这个过程是可数学化表达的。
2. **Reverse（逆向去噪过程）**：训练一个神经网络，学习如何从噪声中逐步恢复原始图像。一旦训练完成，你可以输入一个随机噪声向量，模型就能"反向"生成出一张全新的图像。

### 文本到图像的生成

Text-to-Image 的核心挑战是：如何让生成过程服从文本描述？答案是 **文本编码器（Text Encoder）**。

典型的 Stable Diffusion 架构包含三个组件：

- **Text Encoder**（通常是 CLIP）：将文本 prompt 转化为向量表示（embedding）
- **U-Net + Attention**：在去噪过程中，通过 cross-attention 机制将文本 embedding 注入到图像生成路径中，让模型"看到"文字描述的内容
- **VAE Decoder**：将去噪后的 latent representation 解码为像素级的图像

### Prompt 对图像质量的影响

图像生成中的 prompt engineering 同样重要。几个关键技巧：

- **词序敏感**：越靠前的词权重越高
- **强调关键词**：在一些工具中可用 `()` 或权重语法来强调某个概念
- **风格描述**：加上"photorealistic"、"oil painting"、"anime style"等风格指引
- **艺术家引用**：如"in the style of Studio Ghibli"可以产生特定美学效果

### Negative Prompt（负面提示）

这是一个非常有用的反向技术——告诉模型你**不要**什么。例如：

- Prompt: `a beautiful landscape`
- Negative prompt: `blurry, low quality, distorted, ugly`

负面提示帮助模型避开常见生成缺陷（如手指畸形、画面模糊、构图失衡），**显著提升输出质量**。

## 三、主流图像生成工具

| 工具 | 特点 | 适用场景 |
|------|------|----------|
| **DALL-E 3** | OpenAI 出品，prompt 理解能力极强，无需复杂工程技巧 | 快速原型、概念设计 |
| **Midjourney** | 艺术风格突出，社区生态丰富，通过 Discord 使用 | 艺术创作、概念艺术 |
| **Stable Diffusion** | 开源、可本地部署、可微调、插件生态庞大 | 定制化生成、商业部署 |

三者的核心差异在于：**控制力 vs 易用性**。DALL-E 最容易上手；Midjourney 美学最好；Stable Diffusion 最灵活可控。

---

## 四、本章小结

1. **Prompting 是 GenAI 时代的核心技能**——清晰、具体、给思考时间、用分隔符、用示例，这五条原则适用于所有 LLM 场景。
2. **Diffusion 模型**是当前图像生成的主流范式，其"加噪→去噪"的思想优雅而强大。
3. **Negative prompt** 是一个极易被忽视但效果显著的小技巧。
4. 不同工具各有优劣，选择取决于具体需求——没有"最好的工具"，只有"最适合的工具"。
