# 什么是生成式AI与工作原理

> 吴恩达《Generative AI for Everyone》第一章学习笔记

## 从"理解"到"创造"

传统 AI 擅长分类、预测和推荐——它"看"数据，做出判断。而 **Generative AI（生成式人工智能）** 迈出了关键一步：它不仅能理解，还能**生成**。高质量的文本、图像、音频、甚至代码，都可以由 GenAI 系统直接从零创造出来。

ChatGPT 是其中最具代表性的产品。自 2022 年底发布以来，ChatGPT 以惊人的速度进入了大众视野，也让文本生成成为目前影响最广泛的 GenAI 应用方向。你让它写一封邮件、总结一篇文章、构思一个故事——它都能在几秒内给出一个相当不错的初稿。

## 底层技术：Transformer 与"预测下一个词"

GenAI 并不是凭空变出来的魔法。它的核心技术是 **Transformer 架构**，由 Google 在 2017 年提出。Transformer 的核心创新是 **注意力机制（Attention）**——它让模型能够在处理文本时"关注"到上下文中每个词之间的关系，而不是像之前的 RNN 那样逐词顺序处理。

在这个架构之上，训练过程其实出奇地简单（在概念层面）：**Predict the next word**（预测下一个词）。模型被喂入海量的文本数据，它的任务只有一个——猜下一个词是什么。一次又一次地预测，修正，再预测。当数据量达到十亿、百亿甚至万亿词级别时，一个令人惊讶的现象发生了：模型不仅学会了语法和词汇，还学会了推理、类比、甚至幽默。

这个过程中的最小单位叫 **Token**，可以理解为一个"词块"。Token 是模型理解和生成文本的基本单元，一个英文单词可能拆成 1-2 个 token，一个中文汉字大约是 1-2 个 token。**Context Window（上下文窗口）** 则决定了模型一次能"看到"多少 token——也就是它的短期记忆容量。当前主流模型的 context window 从几千到几十万 token 不等。

## LLM 的两副面孔：Base vs Instruction Tuned

吴恩达在课程中特别区分了两种类型的大语言模型：

**Base LLM（基础模型）**：经过大规模预训练后，模型知道的只是"文本中下一个最可能的词是什么"。你问它"法国的首都是什么？"，它可能会继续补全"法国的首都是一个美丽的城市……"——因为它训练数据里确实有这种描述。它不是在"回答"，而是在"续写"。

**Instruction Tuned LLM（指令微调模型）**：在基础模型之上，通过精心设计的指令数据集进一步训练，让模型学会"人类问问题→给出答案"的模式。同样的问题，它会回答"法国的首都是巴黎。"——这才是我们想要的。

这就是为什么现在的 ChatGPT 看起来像在"听你的话"——它不是天生如此，而是被刻意训练成这样的。

## LLM 训练三阶段

整个训练过程可以概括为三个阶段：

1. **预训练（Pre-training）**：在海量互联网数据上做 next token prediction。这个阶段成本最高，需要数千张 GPU 运行数周甚至数月。产出是 Base LLM。
2. **指令微调（Instruction Tuning）**：用高质量的人类标注数据（问题-答案对）对 Base LLM 进行微调，使其学会遵循指令。产出是一个能"对话"的模型。
3. **RLHF（Reinforcement Learning from Human Feedback）**：人类对模型的多个输出进行偏好排序，训练一个奖励模型，再通过强化学习让模型学会生成更符合人类偏好的回答。这一步让模型的回答更加"有用且无害"。

这三个阶段层层递进，将一个只会"续写"的文本引擎，变成了一个可以协作的智能助手。

## 个人理解

学习这一章最让我震撼的一点是：**整个 GenAI 大厦的根基，竟然只是一个简单的"预测下一个词"任务。** 就像你把一个小孩扔进图书馆让他自己读，读着读着他不仅学会了语言，还学会了思考。这种"大力出奇迹"的涌现现象，本身就是 AI 研究中最迷人的课题之一。

同时，理解 Base LLM vs Instruction Tuned LLM 的区别非常重要——它解释了为什么 LLM 有时候看起来"傻"：不是模型不够强，而是我们用了错误的方式与它交互。这也是 Prompt Engineering 之所以重要的根本原因。
